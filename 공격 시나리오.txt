1. 로그인을 시도.
  - 보통 admin으로 아이디가 있는 경우가 많으니 admin을 아이디로 두고, 비밀번호에 '나 "를 입력해 sql error구문이
출력되는지 확인.
  - '에서 sql error 구문이 출력됐으니, sql injection이 가능하다고 판단 후 시도
  - admin권한의 계정에 로그인 성공

2. search.jsp에서 검색을 함.
  - 검색 결과가 없으니 검색어를 그대로 출력하는 것을 발견함.
  - xss 취약점 공격이 가능하다고 생각해서 <script>alert()</script>를 검색해봄
  - 알림창이 뜨면서 xss 취약점 공격 성공

3. 유저정보 중 데이터베이스에 접근하는 두 가지 메뉴를 발견함
  - sql injection을 시도했지만 통과되지 않음.
  - 그 중 queryxpath.jsp에서도 '와 "를 차례로 검색해봄.
  - '를 검색했을 때와 달리 "를 검색했을 때는 예외처리가 되지 않음을 발견.
  - 이를 이용하여 xss 취약점 공격을 하기 위해 " <script>alert()</script>를 검색함.
  - 검색창 옆에 <script>를 제외한 문구가 등록됨.
  - <script><script>alert()</script>로 수정해서 다시 검색함.
  - 알림창이 뜨면서 xss 취약점 공격 성공
  - sql injection도 가능할까 싶어 시도해봤지만 자바스크립트 상의 취약점임으로 쿼리문이 돌아가지 않을 것으로 판단.
